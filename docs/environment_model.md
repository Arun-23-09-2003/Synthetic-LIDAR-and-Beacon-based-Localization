# Environment Model

Indoor maps are wall-segment floor plans generated by `src/simulation/create_indoor_map.m`.

## Main Functions

- `create_indoor_map.m`: creates outer boundaries, interior walls, wall thicknesses, material factors, labels, bounds, and room labels.
- `src/geometry/segment_intersection.m`: computes segment intersections.
- `src/geometry/segment_clear_of_walls.m`: checks whether a path segment crosses or skims wall geometry.
- `src/geometry/trajectory_wall_crossings.m`: validates generated truth trajectories against wall segments.
- `src/geometry/wall_attenuation.m`: computes hidden simulator-side attenuation between a robot and beacon.
- `src/geometry/raycast_map.m`: casts LiDAR rays against wall segments.

## Inputs

- `scenario_cfg.map_type`
- `scenario_cfg.bounds`
- Wall layout logic selected from:
  - `open_hallway`
  - `multi_room`
  - `large_dense`

## Outputs

The saved `map` structure includes:

- `map.walls`
- `map.bounds`
- `map.wall_thickness_values`
- `map.room_labels`
- `map.name`

Each wall includes:

- `x1`, `y1`, `x2`, `y2`
- `thickness`
- `material_factor`
- `label`

## Important Assumptions

Walls are line segments with thickness and material metadata. Openings are modeled by splitting wall segments and leaving gaps. The filter does not receive wall crossing counts or hidden attenuation values.

The dense map includes an explicit upper doorway aligned with the north corridor so the configured return leg is physically reachable without crossing a wall.
